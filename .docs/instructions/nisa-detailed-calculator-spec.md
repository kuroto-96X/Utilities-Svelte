# NISA運用年率計算ツール（詳細版）実装指示

## 概要

nisa-simple-calculatorは「投資額が常に一定」という前提でModified Dietz法による近似計算を行うのに対し、詳細版は「投資額が途中で変わった（増額・減額した）」場合に対応し、XIRR（内部収益率、ニュートン法による反復計算）で正確な年率を求める。

入力方式は、年ごとの投資額テーブルではなく、「投資記録（開始年月・頻度・投資額）」を任意の数だけ追加できる方式とする。各記録は「その開始年月から、次の記録の開始年月の前まで、その頻度・金額で投資を継続した」とみなして処理する。

フロントエンドのみで完結する想定（サーバーサイド計算不要）。モバイル表示を優先したレイアウトとする。nisa-simple-calculator・nisa-accumulation-simulatorとデザイン（カードのスタイル、スライダー、配色）を統一する。

---

## 入力項目

### 投資記録（リスト、1件以上、追加・削除可能）

各記録は以下の3項目を持つ。

| 項目 | 型 | デフォルト値（初期表示） |
|---|---|---|
| 開始年月 | `month`入力 | 1件目: 任意の過去の年月 / 2件目以降: 任意 |
| 頻度 | セレクト（毎月 / 毎年） | 毎月 |
| 投資額（円） | 数値入力 | 任意 |

初期表示として2件の記録をサンプルとして表示する（例: 1件目「2022年1月・毎月・10,000円」、2件目「2024年4月・毎月・30,000円」）。これは「途中で投資額を増額した」ケースを表す例。

「＋ 投資記録を追加」ボタンで記録を追加できる。各記録には削除ボタン（×）を表示するが、記録が1件のみの場合は削除できない（最低1件は必須）。

### 現在の評価額（円）

数値入力。投資記録の後、フォームの下部に独立して配置する。

### 参考年率（%）

スライダー（0〜15、step 0.5、デフォルト6.0）。将来予測テーブルの比較用シナリオ。

## 計算ロジック

### 1. 投資記録の並び替え

入力された投資記録を、開始年月の昇順にソートする。

### 2. キャッシュフローの生成

ソート済みの記録 `records[0..n-1]` に対して、各記録 `records[i]` ごとに以下を行う。

```
periodStart = records[i].開始年月
if i+1 < n:
    periodEnd = records[i+1].開始年月（この月は含まない、排他的）
else:
    periodEnd = 今月の翌月（つまり今月を含む）

if records[i].頻度 == 毎月:
    periodStart から1ヶ月ずつ進め、日付 < periodEnd である限り、
    各月について {date: その月, amount: records[i].投資額} をキャッシュフローに追加する
else（毎年）:
    periodStart と同じ月で、1年ずつ進め、日付 < periodEnd である限り、
    各年について {date: その年月, amount: records[i].投資額} をキャッシュフローに追加する
```

最終的に、全期間にわたる「投資額のキャッシュフロー一覧」（日付＋金額のリスト）が得られる。

### 3. XIRRによる年率の算出

現在の評価額を `V`、各キャッシュフローを `(date_i, amount_i)`、現在日時を `now` とする。各キャッシュフローについて、現在までの経過年数を以下で求める。

```
yearsAgo_i = (now - date_i) / 365日
```

求めたい年率 `r` は、以下を満たす値である。

```
Σ ( amount_i * (1+r)^yearsAgo_i ) - V = 0
```

これをニュートン法で解く。

```
f(r)  = Σ ( amount_i * (1+r)^yearsAgo_i ) - V
f'(r) = Σ ( amount_i * yearsAgo_i * (1+r)^(yearsAgo_i - 1) )

初期値: r = 0.1
繰り返し（最大100回）:
    if |f'(r)| が極小なら中断
    step = f(r) / f'(r)
    step を [-1, 1] にクランプ（発散防止）
    r_new = r - step
    if |r_new - r| < 1e-8: r = r_new; 終了
    r = r_new
    if r < -0.99: r = -0.99（全損を下回らないようにクランプ）
```

収束しない・`NaN`になった場合は `r = 0` として扱い、結果表示エリアに「計算できませんでした。入力値をご確認ください」等のメッセージを出す。

### 4. 累計投資額・運用期間

```
累計投資額 = キャッシュフローのamountの合計
運用期間（年） = (now - 最も古いキャッシュフローの日付) / 365日
```

### 5. 直近の投資ペース

ソート済み記録の最後の要素（`records[n-1]`）の `頻度` と `投資額` を「現在も継続している投資ペース」として将来予測に使用する。

```
periodsPerYear = 頻度が毎月なら12、毎年なら1
perPeriod = records[n-1].投資額
```

### 6. 将来価値の計算（nisa-simple-calculatorと同じ式）

```
rPeriod = (1 + rateAnnual)^(1/periodsPerYear) - 1
M = years * periodsPerYear

futureValue(rateAnnual, years):
  if M <= 0: return 現在の評価額
  if |rPeriod| ≈ 0: return 現在の評価額 + perPeriod * M
  else: return 現在の評価額 * (1+rPeriod)^M + perPeriod * (((1+rPeriod)^M - 1) / rPeriod)
```

### 7. NISA生涯投資枠（1,800万円）満額到達予測（nisa-simple-calculatorと同じ考え方）

```
NISA_MAX = 18,000,000円
残り投資可能額 = NISA_MAX - 累計投資額

if 残り投資可能額 <= 0:
    満額到達済み（「満額達成済み」と表示。futureValueはyears=0で計算）
else:
    満額到達までの期数 = 残り投資可能額 / perPeriod
    満額到達までの年数 = 満額到達までの期数 / periodsPerYear
```

## 出力項目

### 計算結果（メトリックカード、2x2グリッド）

- 年率（XIRR）（%、小数1桁）
- 運用期間（年、小数1桁）
- 累計投資額（円、3桁区切り）
- 直近の投資ペース（例: 「30,000円 / 月」）

### 将来予測テーブル（単位: 万円）

nisa-simple-calculatorと同一の構成。

| 行 | ご自身の年率での予測 | 参考年率での予測 |
|---|---|---|
| 5年後 | futureValue(r, 5) | futureValue(参考年率, 5) |
| 10年後 | futureValue(r, 10) | futureValue(参考年率, 10) |
| 20年後 | futureValue(r, 20) | futureValue(参考年率, 20) |
| NISA満額到達時（または「満額達成済み」） | futureValue(r, 満額到達までの年数) | 同上 |

表示は万円単位に変換（`Math.round(円 / 10000)`）。

## 画面レイアウト（上から順）

1. **入力カード**
   - 見出し「投資記録」＋ 説明文（「開始年月以降、次の記録の開始年月になるまで、その投資額・頻度を継続したものとして計算します。」）
   - 投資記録のリスト（各記録: 開始年月＋頻度を1行、投資額＋削除ボタンを1行）
   - 「＋ 投資記録を追加」ボタン
   - 区切り線
   - 「現在の評価額（円）」入力

2. **計算結果カード（2x2グリッド）**
   - 年率（XIRR） / 運用期間 / 累計投資額 / 直近の投資ペース

3. **将来予測カード**
   - 参考年率スライダー（0〜15%、step 0.5、デフォルト6%）
   - 将来予測テーブル（5年後 / 10年後 / 20年後 / NISA満額到達時）
   - 免責文

入力値の変更（記録の追加・削除・編集、現在の評価額、スライダー）に応じて、計算結果と将来予測テーブルをリアルタイムに再計算・再表示する。

## エッジケース処理

- **投資記録が1件のみで、開始年月が今月の場合**: キャッシュフローが1件（今月分）のみとなり、`yearsAgo ≈ 0` になる。この場合XIRRの計算が不安定になりやすいため、`yearsAgo` が極小（例: 0.05年未満）のキャッシュフローのみで構成される場合は「投資期間が短すぎるため、年率を正確に計算できません」という旨を表示し、結果カードは非表示にする。
- **現在の評価額が0または極端に小さい場合**: XIRRが大きく負の値（ほぼ-100%）に近づく。`r < -0.99` でクランプしているため表示は「-99.0%」等になるが、それで問題ない。
- **記録の開始年月が重複・前後逆転している場合**: ソート処理で自動的に順序を補正するため、入力順序に依存しない。同じ開始年月が複数ある場合は、ソート後の順序（配列内の元の順序を保持する安定ソート）で処理されるため、実質的に後者の記録の `periodStart` が前者の `periodEnd` と同じになり、前者の期間は0になる（実害はないが、UI上「同じ開始年月の記録が複数あります」という注意書きを表示してもよい）。
- **累計投資額が既にNISA満額（1,800万円）を超えている**: nisa-simple-calculatorと同様、「満額達成済み」と表示し、新規投資分を含めない将来予測を表示する。
- **ニュートン法が収束しない場合**: `r = 0` として扱い、結果カードに「計算できませんでした。入力値をご確認ください」というメッセージを表示する。


## 技術メモ

- すべての計算はクライアントサイドJavaScriptで完結する（サーバー通信不要）。
- 投資記録のリストはDOM上で動的に追加・削除する。各記録の入力要素（month, select, number）にイベントリスナーを設定し、入力変更時に再計算する。
- XIRRのニュートン法は、キャッシュフロー数が数十〜百件程度であれば、毎回の入力変更時に再計算しても性能上問題ない。
- 画面に表示するすべての数値は `Math.round()` または `toFixed()` で丸め、`toLocaleString()` で3桁区切り表示する。
- モバイル幅（〜400px程度）を基準にレイアウトを組み、入力欄・カードは縦並びを基本とする。
- nisa-simple-calculator・nisa-accumulation-simulatorと同一サイト内で相互リンクする想定。デザイン（カードのスタイル、スライダーのスタイル、配色）はnisa-simple-calculator・nisa-accumulation-simulatorと統一する。
