# 画像圧縮・リサイズ・形式変換ツール 実装指示書

## 概要

既存のSvelte(SvelteKit)プロジェクトに、画像の圧縮・リサイズ・形式変換を行うクライアントサイド完結型のWebツールを実装する。サーバーへのアップロードは行わず、すべてブラウザ内(Canvas API)で処理する。広告収益を目的としており、SEOで複数の検索キーワードを狙えるページ構成にする。

## 主要機能要件

### 入力
- ドラッグ&ドロップでの画像ファイル受け取り
- ファイル選択ボタンでの受け取り
- 複数ファイルの同時アップロードに対応
- 対応形式: JPEG, PNG, WebP, GIF(入力のみ可)

### 変換設定
- 出力形式の選択: JPEG / PNG / WebP
- 品質スライダー(10〜100、デフォルト80) ※PNG選択時は無効化または非表示
- リサイズ設定:
  - 幅(px)・高さ(px)の入力フィールド
  - アスペクト比固定のチェックボックス(デフォルトON)
  - 「リサイズしない(元のサイズを維持)」オプション

### 処理・出力
- Canvas APIを使い、選択された形式・品質・サイズで画像を再エンコード
- 変換前・変換後のファイルサイズを表示し、削減率(%)も表示する
- 変換後のプレビュー画像を表示
- 個別ダウンロードボタン
- 複数ファイルの場合は「すべてZIPでダウンロード」ボタンを追加(JSZipを使用)

### エラーハンドリング
- 非対応形式のファイルが選択された場合は分かりやすいエラーメッセージを表示
- 処理中はローディング状態を表示(特に大きな画像・多数のファイルの場合)

## UI/レイアウト

以下の構成で1ページにまとめる(上から順):

1. 広告枠(レスポンシブバナー) - プレースホルダーのdivでよい
2. ページタイトルと簡単な説明文(「ファイルはサーバーに送信されません」という訴求を含める)
3. ドラッグ&ドロップ可能なアップロードエリア(破線ボーダー、アイコン付き)
4. 変換設定エリア(出力形式・品質スライダー・幅/高さ入力をグリッドレイアウトで横並び、レスポンシブ対応)
5. 変換前/変換後の比較表示(2カラム、ファイルサイズと削減率を表示)。複数ファイルの場合はファイルごとにこのカードをリスト表示
6. ダウンロードボタン(個別 / 複数時はZIP一括)
7. 広告枠(記事内レクタングル) - プレースホルダーのdivでよい
8. 使い方の説明文(SEO向けテキスト、200〜400文字程度)

スタイルは既存プロジェクトのデザイントークン・CSS変数があれば優先して使用する。なければシンプルなフラットデザイン(角丸、薄いボーダー、余白を十分に)で実装する。

## 技術要件

- SvelteKit(既存プロジェクト構成に準拠)
- 画像処理: Canvas API(`canvas.toBlob()`)を使用し、形式変換・リサイズ・品質調整を行う
- 複数ファイルのZIP化: JSZip(npm install jszip)
- ファイルダウンロード: Blob URLを生成し`<a download>`で実行
- 状態管理はSvelteのリアクティブ変数(`$state`または`writable`store)で十分。外部の状態管理ライブラリは不要
- 画像処理はメインスレッドのCanvas APIで問題ない(必要であればWeb Workerでの実装も検討可とするが、MVPでは不要)

## ファイル構成案

```
src/routes/
  image-tools/
    +page.svelte          # 汎用ツール本体(全形式対応)
    +page.ts              # SEO用メタデータ(title, description)
    png-to-jpg/
      +page.svelte        # PNG→JPG変換に特化したランディングページ
    jpg-to-webp/
      +page.svelte        # JPG→WebP変換に特化したランディングページ
    compress-image/
      +page.svelte        # 画像圧縮に特化したランディングページ

src/lib/
  components/
    ImageUploader.svelte   # ドラッグ&ドロップ・ファイル選択
    ImageSettings.svelte   # 形式・品質・サイズ設定UI
    ImageComparison.svelte # 変換前/後の比較表示
  utils/
    imageProcessor.ts      # Canvas APIによる変換・リサイズ・圧縮ロジック
    zipDownload.ts          # JSZipを使った一括ダウンロード処理
```

各ランディングページ(`png-to-jpg`など)は、共通コンポーネント(`ImageUploader`, `ImageSettings`, `ImageComparison`)を再利用し、出力形式のデフォルト値とページ固有のSEO用テキスト・メタデータのみを変える形にする。

## SEO考慮事項

- 各ページの`+page.ts`(または`+page.server.ts`)で、ページごとに異なる`title`と`meta description`を設定する
  - 例: `png-to-jpg` → title: 「PNGをJPGに変換 - 無料オンラインツール」
- 各ページ下部に、そのページのキーワードに関連した説明文(200〜400文字程度)を配置する
- 構造化データ(WebApplication schema)のJSON-LDをheadに追加できると望ましい(任意)

## 広告枠

- ページ上部と、変換結果表示後の2箇所にプレースホルダー要素を用意する
- `<div class="ad-slot ad-slot--top">` のようにクラス名をつけ、後でAdSenseなどのスクリプト・タグに差し替えやすい構造にする
- アップロード操作や変換結果の視認性を妨げない位置・サイズにする

## 実装ステップ(目安)

1. `src/lib/utils/imageProcessor.ts` の実装(Canvas APIでのリサイズ・形式変換・品質調整。入出力は File/Blob)
2. `ImageUploader.svelte`(ドラッグ&ドロップ、複数ファイル対応)
3. `ImageSettings.svelte`(出力形式・品質・サイズ設定UI)
4. `ImageComparison.svelte`(変換前/後のプレビューとファイルサイズ比較)
5. `src/routes/image-tools/+page.svelte` で上記コンポーネントを組み合わせ、汎用ツールを完成させる
6. JSZipを使った複数ファイル一括ダウンロード機能の追加
7. SEO用の個別ランディングページ(`png-to-jpg`など)を共通コンポーネントの再利用で追加
8. 広告枠のプレースホルダー設置とレイアウト調整
9. レスポンシブ対応の確認(モバイル表示でのレイアウト崩れがないか)
