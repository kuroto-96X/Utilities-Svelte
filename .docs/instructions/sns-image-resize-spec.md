# SNS用画像サイズ調整ツール 実装指示書

## 概要

既存のSvelte(SvelteKit)プロジェクトに、SNS各サービスの推奨画像サイズにワンクリックでリサイズ・クロップできるクライアントサイド完結型ツールを実装する。サーバーへのアップロードは行わず、すべてブラウザ内で処理する。画像圧縮・変換ツール(`image-tools`)とは別画面とし、アップロード・ダウンロード等の共通ロジックは可能な範囲で共有する。

## 主要機能要件

### プリセット選択
- 主要なSNS向けサイズをカード形式で一覧表示し、クリックで選択する
- 表示数が多くなる場合は「もっと見る」で展開できるようにする
- 選択中のプリセットを視覚的にハイライトする(枠線・背景色)

#### プリセット一覧(初期実装分)

| 名称 | 用途 | サイズ(px) | アスペクト比 |
|---|---|---|---|
| Instagram 投稿(正方形) | Instagram | 1080 × 1080 | 1:1 |
| Instagram 投稿(縦長) | Instagram | 1080 × 1350 | 4:5 |
| Instagram ストーリー / リール | Instagram | 1080 × 1920 | 9:16 |
| X(Twitter) 投稿画像 | X | 1600 × 900 | 16:9 |
| X(Twitter) ヘッダー画像 | X | 1500 × 500 | 3:1 |
| YouTube サムネイル | YouTube | 1280 × 720 | 16:9 |
| YouTube チャンネルアート | YouTube | 2560 × 1440 | 16:9 |
| Facebook カバー画像 | Facebook | 820 × 312 | 約 2.6:1 |
| LinkedIn 投稿画像 | LinkedIn | 1200 × 627 | 約 1.91:1 |

上記のプリセットデータは `src/lib/data/snsPresets.ts` のような定数ファイルにまとめ、名称・サイズ・アスペクト比・アイコン名(Tabler iconsなど)を持つオブジェクト配列として管理する。今後プリセットを追加・編集しやすい構造にする。

### 画像アップロード
- ドラッグ&ドロップ、ファイル選択の両方に対応
- 画像圧縮・変換ツール(`image-tools`)で実装済みのアップロードコンポーネントを再利用・共通化できる場合は流用する

### クロップ操作
- 選択したプリセットのアスペクト比に固定したクロップ枠を画像上に表示する
- ドラッグでクロップ位置を移動できる
- スライダーまたはピンチ操作で拡大・縮小できる
- クロップ操作のUIは自作せず、`svelte-easy-crop`(npm install svelte-easy-crop)の利用を基本方針とする。同ライブラリでカバーできない調整が必要な場合のみ、Canvas APIでの自前実装を検討する

### 出力
- 「ダウンロード」ボタンで、選択中のプリセットの解像度・アスペクト比に合わせて書き出した画像をダウンロードする
- 出力形式はPNGを基本とし、必要に応じてJPEG/WebPの選択肢も追加可能とする(MVPではPNG固定でも可)
- ダウンロードボタンには、書き出されるサイズ(例:「ダウンロード(1080 × 1080 PNG)」)を表示する

## UI/レイアウト

以下の構成で1ページにまとめる(上から順):

1. 広告枠(レスポンシブバナー) - プレースホルダーのdivでよい
2. ページタイトルと説明文(「サーバーには送信されません」という訴求を含める)
3. プリセット選択エリア(カードのグリッド表示、選択中のものをハイライト)
4. アップロードエリア(ドラッグ&ドロップ対応)
5. クロップ編集エリア(選択したアスペクト比でのクロップUI、拡大スライダー付き)
6. ダウンロードボタン(書き出しサイズを明記)
7. 広告枠(記事内レクタングル) - プレースホルダーのdivでよい
8. 対応プリセットの説明文(SEO向けテキスト、200〜400文字程度)

スタイルは既存プロジェクトのデザイントークン・CSS変数があれば優先して使用する。画像圧縮・変換ツールと共通のレイアウトコンポーネント(ヘッダー・フッター・他ツールへのナビゲーション)があれば、それを使い回す。

## 技術要件

- SvelteKit(既存プロジェクト構成に準拠)
- クロップUI: `svelte-easy-crop`
- クロップ結果の画像生成: Canvas API(`canvas.toBlob()`)
- ファイルダウンロード: Blob URLを生成し`<a download>`で実行
- 状態管理はSvelteのリアクティブ変数(`$state`または`writable`store)で十分

## ファイル構成案

```
src/routes/
  sns-image-resize/
    +page.svelte          # SNS用画像サイズ調整ツール本体
    +page.ts              # SEO用メタデータ(title, description)

src/lib/
  data/
    snsPresets.ts          # プリセット一覧(名称・サイズ・アスペクト比・アイコン)
  components/
    ImageUploader.svelte   # 画像圧縮・変換ツールと共通化(既存があれば再利用)
    PresetSelector.svelte  # プリセット選択カードグリッド
    CropEditor.svelte      # svelte-easy-cropを使ったクロップUI
  utils/
    cropToBlob.ts           # クロップ結果をCanvas経由でBlobに変換する処理
```

`ImageUploader.svelte` が画像圧縮・変換ツール側に既に存在する場合は、そのコンポーネントをこのツールでもimportして再利用する。存在しない場合は、両ツールで共通利用できる形で新規実装し、`src/lib/components/` に配置する。

## SEO考慮事項

- `+page.ts`(または`+page.server.ts`)で、ページ専用の`title`と`meta description`を設定する
  - 例: title: 「SNS画像サイズ変換 - Instagram・X・YouTube対応の無料ツール」
- ページ下部に、対応プリセットや使い方についての説明文(200〜400文字程度)を配置する
- 画像圧縮・変換ツールのページとの間に内部リンク(「画像を圧縮したい場合はこちら」等)を設置し、サイト内の回遊を促す

## 広告枠

- ページ上部と、クロップ編集エリアの下(ダウンロードボタンの後)の2箇所にプレースホルダー要素を用意する
- クラス名は画像圧縮・変換ツールと統一する(例: `ad-slot ad-slot--top`, `ad-slot ad-slot--in-content`)ことで、後で広告タグを差し替える際に一括対応しやすくする

## 実装ステップ(目安)

1. `src/lib/data/snsPresets.ts` にプリセットデータを定義
2. `svelte-easy-crop` の導入と、`CropEditor.svelte` での基本的なクロップ表示の実装
3. `PresetSelector.svelte` の実装(プリセット選択時にアスペクト比が`CropEditor`に反映されるようにする)
4. アップロード機能の実装(既存コンポーネントの再利用、または新規実装)
5. クロップ結果を指定解像度のPNGとして書き出す処理(`cropToBlob.ts`)とダウンロード機能の実装
6. `src/routes/sns-image-resize/+page.svelte` で各コンポーネントを組み合わせ、全体のレイアウトを完成させる
7. SEO用メタデータの設定、説明文の追加、画像圧縮・変換ツールへの内部リンク設置
8. 広告枠のプレースホルダー設置とレイアウト調整
9. レスポンシブ対応の確認(モバイルでのクロップ操作のしやすさを含む)
