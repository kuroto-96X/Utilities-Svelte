# 積立シミュレーションツール（②）実装指示

## 概要

毎月または毎年の積立投資額・投資期間・想定年利を入力すると、最終的な評価額・累計投資額・損益額・損益率を計算し、資産の推移をグラフで表示するWebツール。

①（NISA運用年率計算ツール）の逆パターンにあたるが、①との計算ロジックの整合性は考慮しない。標準的な年金（積立）の将来価値公式をそのまま使用する。

フロントエンドのみで完結する想定（サーバーサイド計算不要）。モバイル表示を優先したレイアウトとする。

---

## 入力項目

| 項目 | 型 | デフォルト値 | 説明 |
|---|---|---|---|
| 投資頻度 | トグルボタン（毎月 / 毎年） | 毎月 | 積立の頻度。選択に応じて下の入力ラベルが「毎月の投資額（円）」/「毎年の投資額（円）」に切り替わる |
| 1回あたりの投資額（円） | 数値入力 | 30000 | 毎月または毎年投資する金額 |
| 投資期間（年） | スライダー（1〜40、step 1） | 20 | 積立を継続する年数 |
| 想定年利（%） | スライダー（0〜15、step 0.5） | 5.0 | 想定する年率。ユーザーが自由に変更可能 |

---

## 計算ロジック

### 1. 期数の算出

```
periodsPerYear = 毎月なら12, 毎年なら1
n = 投資期間（年） * periodsPerYear
```

### 2. 期間レートへの変換

年率を、積立の頻度に合わせた期間レートに変換する。

```
rPeriod = (1 + 想定年利/100)^(1/periodsPerYear) - 1
```

### 3. 将来価値（積立の年金終価）

`numPeriods` 期後の評価額を計算する関数として定義する。

```
fv(numPeriods):
  if numPeriods == 0: return 0
  if rPeriod ≈ 0:      return 1回あたりの投資額 * numPeriods
  else:                return 1回あたりの投資額 * ((1+rPeriod)^numPeriods - 1) / rPeriod
```

`rPeriod ≈ 0` の判定は `Math.abs(rPeriod) < 1e-9` 程度の許容誤差で行う（想定年利0%のときに0除算を避けるため）。

### 4. 最終結果

```
最終評価額 = fv(n)
累計投資額 = 1回あたりの投資額 * n
損益額     = 最終評価額 - 累計投資額
損益率(%)  = 累計投資額 > 0 ? (損益額 / 累計投資額) * 100 : 0
```

### 5. 資産推移グラフ用データ

`y = 0` から `投資期間（年）` まで、1年刻みで以下を計算する。

```
for y = 0 to 投資期間（年）:
  numPeriods = y * periodsPerYear
  累計投資額(y) = 1回あたりの投資額 * numPeriods
  評価額(y)     = fv(numPeriods)
```

---

## 出力項目

### 計算結果（メトリックカード、2x2グリッド）

- 最終評価額（円、3桁区切り）
- 累計投資額（円、3桁区切り）
- 損益額（円、3桁区切り。マイナスの場合は負号付き表示）
- 損益率（%、小数1桁。マイナスの場合は負号付き表示）

### 資産推移グラフ

- 横軸: 経過年数（0年 〜 投資期間（年））
- 縦軸: 金額（万円換算）
- 「評価額」の推移を実線＋エリア塗り（緑系: `#0F6E56` 線 / `#1D9E75` 塗り、不透明度0.15）で表示
- 「累計投資額」の推移を破線（グレー系: `#888780`）で重ねて表示
- 凡例: グラフ上部に「評価額」「累計投資額」の色見本付きラベルを表示
- 軸ラベル: 横軸の開始（0年）・終了（投資期間の年数）、縦軸の最大値（万円）を表示する程度で十分（細かい目盛りは不要）

## 画面レイアウト（上から順）

1. **入力カード**
   - 投資頻度トグル（毎月 / 毎年）
   - 1回あたりの投資額（円）の数値入力（ラベルは頻度に応じて「毎月の投資額」/「毎年の投資額」に動的に切り替える）
   - 投資期間（年）スライダー（1〜40、デフォルト20）
   - 想定年利（%）スライダー（0〜15、step 0.5、デフォルト5.0）

2. **計算結果カード（2x2グリッド）**
   - 最終評価額 / 累計投資額 / 損益額 / 損益率

3. **資産推移グラフカード**
   - 凡例（評価額 / 累計投資額）
   - SVGまたはCanvasによる折れ線・エリアグラフ
   - 免責文（下記参照）

すべての入力値・スライダー操作で、計算結果とグラフをリアルタイムに再計算・再描画する。

---

## エッジケース処理

- **想定年利が0%の場合**: `rPeriod ≈ 0` となるため、`fv()` は単純な線形計算（投資額×期数）にフォールバックする。グラフ上は「評価額」と「累計投資額」が完全に一致する直線になる。
- **1回あたりの投資額が0または未入力の場合**: 全ての結果を0として表示し、エラーにはしない。
- **損益率の計算で累計投資額が0の場合**: 0除算を避けるため、損益率は0%として表示する。
- **投資期間スライダーの最大値（40年）変更時**: グラフのx軸ラベル・スケールも合わせて再計算する。

## 技術メモ

- すべての計算はクライアントサイドJavaScriptで完結する（サーバー通信不要）。
- グラフはSVGの`<path>`要素を動的に生成する方式で十分（外部グラフライブラリは必須ではない）。
- 画面に表示するすべての数値は `Math.round()` または `toFixed()` で丸め、`toLocaleString()` で3桁区切り表示する。
- モバイル幅（〜400px程度）を基準にレイアウトを組み、入力欄・カードは縦並びを基本とする。
- ①（NISA運用年率計算ツール）と②（本ツール）は将来的に同一サイト内で相互リンクする想定。デザイン（カードのスタイル、スライダーのスタイル、配色）は①と統一する。
