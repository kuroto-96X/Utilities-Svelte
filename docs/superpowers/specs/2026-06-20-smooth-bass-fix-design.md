# 設計ドキュメント: スムーズベース再生の修正

## 1. 目的

コード進行プリセットで「スムーズベース（転回形）」を有効にしたとき、上部の共通転回形設定に左右されず、プリセット固有のボイシングが実際の再生音へ適用されるようにする。また、ループ終端から先頭へ戻る箇所を含め、既知の大きなベース跳躍を抑える。

## 2. 現状と原因

`ProgressionPlayer.svelte` は、スムーズベース用データが配列ならそのオクターブ補正を使うが、`null` なら共通の `inversion` を使う。このため、チェック中でも共通転回形が一部ステップに混ざる。

さらに、以下のプリセットは各周期内の移動量は減っているものの、ループ境界で大きく跳ぶ。

- 基本循環: `C5 → F4 → E4 → D4 → C5`（境界で10半音）
- 50年代進行: `C4 → E4 → F4 → G4 → C4`（境界で7半音）
- ジャズ循環: `C4 → E4 → F4 → G4 → C4`（境界で7半音）

## 3. 検討した方式

### A. ボイシングデータだけを修正する

変更量は小さいが、`null` が共通転回形へフォールバックする構造は残る。再発防止にならないため採用しない。

### B. 再生コンポーネント内の条件分岐だけを修正する

共通転回形との混在は直せるが、Svelteコンポーネント内部にボイシング規則が埋まり、実MIDIを単体テストしにくい。

### C. 純粋なボイシング解決関数を追加し、必要なデータだけ調整する（採用）

通常再生とスムーズ再生の規則を純粋関数に集約する。スムーズ再生時の `null` は「補正なし＝ルート形」と明示的に扱い、共通転回形を参照しない。既知の3プリセットだけデータを最小修正する。

## 4. 実装設計

### 4.1 ボイシング解決

`scaleData.ts` に、コードのインターバルを再生用インターバルへ変換する純粋関数を追加する。

```ts
resolveProgressionVoicing(intervals, smoothVoicing, useSmoothedBass, inversion)
```

- スムーズベースOFF: 既存どおり `applyInversion(intervals, inversion)` を返す。
- スムーズベースON: 各構成音へ `smoothVoicing` のオクターブ補正を加える。
- スムーズベースONかつ `smoothVoicing === null`: 各補正を0として扱い、必ずルート形を返す。
- 入力配列は変更しない。

`ProgressionPlayer.svelte` のダイアトニック進行とクロマティック進行は、どちらもこの関数の戻り値からMIDIを生成する。

### 4.2 ループ境界の調整

Cメジャー時の最低音を以下へ変更する。

- 基本循環: `C4 → F4 → E4 → D4 → C4`（最大5半音）
- 50年代進行: `C4 → E4 → F4 → D4 → C4`（最大4半音）
- ジャズ循環: `C4 → E4 → F4 → D4 → C4`（最大4半音）

変更する `smoothVoicings` は次の3箇所だけとする。

- 基本循環の先頭I: `[12,12,12]` → 明示的なルート形 `[0,0,0]`
- 50年代進行の末尾V: `null` → 第2転回形 `[0,0,-12]`
- ジャズ循環の末尾V: `null` → 第2転回形 `[0,0,-12]`

## 5. テスト設計

実装前に以下の失敗テストを追加する。

1. スムーズベースONでは、`smoothVoicing === null` でも共通転回形に依存しない。
2. スムーズベースOFFでは、従来どおり共通転回形が適用される。
3. Cメジャーの基本循環・50年代進行・ジャズ循環が、ループ境界込みで最大5半音以内になる。
4. 全プリセットで `smoothVoicings` のステップ数が進行ステップ数と一致する。
5. 全既存テスト、型検査、ビルドを通す。
6. ブラウザでチェックON時の実Web Audio入力MIDIを採取し、共通転回形を変えても同一になることと、3進行の最低音列を確認する。

## 6. 変更範囲

- `src/lib/scaleData.ts`
- `src/lib/scaleData.test.ts`
- `src/lib/components/ProgressionPlayer.svelte`

UI文言・レイアウト・他の再生機能は変更しない。

## 7. 受け入れ条件

- スムーズベースON時の実再生MIDIが共通転回形設定に依存しない。
- 基本循環・50年代進行・ジャズ循環のループ境界を含む最低音移動が最大5半音以内である。
- スムーズベースOFF時の転回形動作は変わらない。
- コードのピッチクラスは変更されず、変わるのはオクターブ配置だけである。
- 既存のコード進行停止・切替・ハイライト動作を維持する。
