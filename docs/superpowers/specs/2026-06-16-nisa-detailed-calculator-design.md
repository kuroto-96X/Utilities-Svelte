# NISA詳細年率計算ツール 設計

## 目的

途中で投資額や頻度が変わったNISA運用について、投資記録と現在評価額からXIRRによる年率を算出する。単純版の「投資額一定」前提では扱いにくい増額・減額ケースを、モバイル優先の入力UIで扱えるようにする。

## アプローチ

計算ロジックは `src/lib/nisa-detailed-calculator.ts` に分離し、ページ側は入力状態と表示に集中させる。既存の `nisa-simple-calculator` と `nisa-accumulation-simulator` に合わせて、白いカード、teal系アクセント、2列メトリック、range入力を使う。

ルートは `/nisa-detailed-calculator/` とし、`site.ts` の投資カテゴリに `NISA詳細年率計算(開発中)` として追加する。

## UI

入力カードには投資記録リスト、追加ボタン、現在評価額入力を置く。各投資記録は開始年月、頻度、投資額、削除ボタンを持つ。初期値は 2022-01 毎月 10,000円、2024-04 毎月 30,000円、現在評価額 1,500,000円 とする。

計算結果カードは、年率、運用期間、累計投資額、直近の投資ペースの2x2グリッドにする。投資期間が短すぎる場合やXIRRが収束しない場合は、仕様の文言に沿った注意メッセージを表示する。

将来予測カードには参考年率スライダーと、5年後、10年後、20年後、NISA満額到達時の比較テーブルを置く。単位は万円で丸める。

## 計算

投資記録は開始年月の昇順に安定ソートする。各記録は、次の記録の開始年月の前月まで有効とみなし、毎月または毎年のキャッシュフローを生成する。最後の記録は今月まで継続しているものとして扱う。

XIRRは仕様どおりニュートン法で解く。発散防止としてstepを `[-1, 1]` に制限し、年率は `-0.99` を下限にする。収束しない場合は年率0%を返し、UIに計算不能メッセージを出す。

将来価値は現在評価額に直近の投資ペースを積み増す式で計算する。累計投資額が1,800万円以上の場合は「満額達成済み」として、新規投資を含めない予測を表示する。

## エラー処理

入力不足、投資額0以下、現在評価額が数値でない場合は結果を出さず、入力確認メッセージを表示する。キャッシュフローがない場合も同じ扱いにする。

すべてのキャッシュフローが現在から0.05年未満の場合は、投資期間が短すぎる旨を表示し、年率メトリックと予測テーブルは表示しない。同じ開始年月が複数ある場合は、注意文を入力カード下に表示する。

## テスト

`src/lib/nisa-detailed-calculator.test.ts` を追加し、以下を確認する。

- 投資記録の切り替え月が排他的に扱われること
- 毎年頻度のキャッシュフローが年単位で生成されること
- 評価額が累計投資額と同程度なら年率が大きく外れないこと
- 投資期間が短すぎる場合にエラーになること
- NISA満額到達済みの予測が達成済み表示になること

## 完了条件

`npm run test`、`npm run check`、`npm run build` を通す。最後に `npm run preview` で `/nisa-detailed-calculator/` を確認する。
