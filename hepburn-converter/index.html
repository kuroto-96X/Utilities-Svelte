カタカナは同じルールで変換されます。外来語・特殊音および小書き仮名フォールバックはカタカナにも適用。

変換ルール設定について

プリセット

パスポート式（デフォルト） — 長音省略 / mn 使い分け / 区切りなし

鉄道式 — マクロン（ā, ō 等）/ mn 使い分け / ハイフン（n-）

道路標識式 — 長音省略 / 常に n / 区切りなし

カスタム — 下記の設定を自由に組み合わせ。別プリセットへ切り替えて再びカスタムに戻すと直前の値が復元されます。

長音

表記しない — おう・うう等の長音を省略。例: とうきょう → tokyo

マクロン — 伸ばす母音の上にバーを付加。例: とうきょう → tōkyō

母音を重ねる — 母音を繰り返す。例: とうきょう → toukyou

撥音「ん」

b/m/p 前はm — ん の後に b, m, p が続く場合は m に変換。例: しんぶん → shimbun

常に n — すべて n で表記。例: しんぶん → shinbun

撥音+母音/y

何もしない — 区切りを入れない。例: しんよう → shinyou

アポストロフィ — アポストロフィで区切る。例: しんよう → shin'you

ハイフン — ハイフンで区切る。例: しんよう → shin-you

半角 / 全角
出力文字の幅を選択します。全角ではローマ字が全角文字（Ａ, ｂ 等）で出力されます。「単語区切り」のスペースも全角スペースになります。
出力形式

PascalCase — 形態素解析またはスペースで区切られた各単語の先頭を大文字に

小文字 — すべて小文字

大文字 — すべて大文字

ヴ行

va / vi / vu / ve / vo — v を使った標準ヘボン式表記

ba / bi / bu / be / bo — b に置き換える表記（実際の発音に近い）

bua / bui / bu / bue / buo — bu + 母音 で表す表記

形態素解析 / 単語区切り
形態素解析を使うと日本語の単語境界を自動検出し、PascalCase での先頭大文字処理や「単語区切り」のスペース挿入に反映されます。単語区切りのスペースは形態素解析が有効なときのみ機能します。
漢字変換
形態素解析で漢字を読み（カタカナ）に変換してからローマ字化します。形態素解析が有効な場合は単語ごとに変換されます。

その他の仕様

  • 文字数制限: 10,000文字。超過すると自動変換を停止します（変換ボタンは引き続き使用可）。
  • 入力中は 300ms 後に自動変換されます。IME（日本語入力）の変換確定前は変換しません。
  • 設定変更時は自動変換されません。変換ボタンを押すと最新の設定で変換されます。
  • スペース（半角・全角とも）は文の区切りとして扱い、それぞれを独立して変換します。PascalCase では各文の先頭が大文字になります。
  • 漢字変換が無効な場合、漢字はそのまま出力されます。数字・ASCII記号は常にそのまま出力されます。
  • 変換できない文字（絵文字など）はそのまま出力され、警告が表示されます。
