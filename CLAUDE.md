# プロジェクト規約

## ブランチ運用

- 作業ブランチ（例: `feat-*`）でのみコミットする
- `master` へのマージ・プッシュは行わない（ユーザーが手動で行う）

## 作業完了前の確認

コードを書き終えたら、以下を必ず実行してから完了とする。

```bash
# 1. ビルドが通ることを確認
npm run build

# 2. 開発サーバーを起動してブラウザで動作確認
npm run dev
# → http://localhost:5173 を開いて確認
```

ビルドエラーや画面の崩れがあれば、完了報告の前に修正する。

## 新しいツール（ページ）の追加

上部メニューに追加するリンク名には必ず `(開発中)` を付ける。

例: `ヘボン式変換` → `ヘボン式変換(開発中)`
